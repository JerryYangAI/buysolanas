## 1. 项目概述

基于五大核心策略，将 BuySolanas.com 从一个基础介绍页升级为中文圈最权威的 Solana 教育与信息平台。

### 核心定位

**新口号：「从零到 Solana，你的 Web3 第一站」**

替换原有口号"安全学习Solana"的原因：
- 原口号过于防御性，"安全"给用户带来负面联想
- 缺乏行动感，未清晰表达网站核心价值
- 新口号明确目标用户（零基础），强调一站式价值主张

### 核心增长飞轮

```
词汇表 + 课程内容
        ↓
    SEO 自然流量
        ↓
  Newsletter 订阅转化
        ↓
    Discord 社区沉淀
        ↓
  UGC 内容 → 新流量
```

### 方案规模

| 维度 | 数量 |
|------|------|
| 核心策略 | 5 个 |
| 开发阶段 | 4 个 |
| 预计工期 | 约 16 周 |
| 功能模块 | 20+ 个 |

### 策略一：实时行情 Ticker

> **优先级：P0 | 工期：2–3 周 | 影响：高 | 开发量：低**

参照 CoinGecko 顶部价格 bar，接入实时 SOL 数据，让用户每次访问都能看到最新行情，提升网站的"活跃感"与可信度。

#### 1.1 Ticker Bar 显示字段

- SOL/USDT 实时价格
- 24h 涨跌幅（绿色/红色 + 上下箭头区分）
- 24h 交易量
- 市值（Market Cap）
- Solana TPS 实时值
- 全球加密市值总量

#### 1.2 行情详情页内容

- K 线图（1D / 7D / 1M / 1Y）
- 持仓地址数量趋势
- SOL 生态 Top 10 代币
- 质押 APY 实时数据
- Solana 验证节点数
- 历史价格与关键事件

#### 1.3 数据来源方案

| 方案 | 说明 | 成本 |
|------|------|------|
| CoinGecko Free API | 每分钟 30 次调用，覆盖主要价格数据 | 免费 |
| Birdeye API | Solana 专属链上数据，更丰富 | 免费/付费 |

> **架构建议：** 生产环境在后端做 Redis 缓存，每 30 秒刷新一次，避免前端直接调用导致限流。

#### 1.4 Python 后端缓存示例

```python
import requests, redis, json

def get_sol_price():
    r = redis.Redis()
    cached = r.get('sol_price')
    if cached:
        return json.loads(cached)

    url = "https://api.coingecko.com/api/v3/simple/price"
    params = {
        "ids": "solana",
        "vs_currencies": "usd,cny",
        "include_24hr_change": "true",
        "include_market_cap": "true",
        "include_24hr_vol": "true"
    }
    data = requests.get(url, params=params).json()
    r.setex('sol_price', 30, json.dumps(data))  # 缓存 30 秒
    return data
```

#### 1.5 前端实现要点

- Ticker bar 固定在 nav 下方，sticky 定位
- 价格涨跌用绿色/红色 + 上下箭头区分
- 每 30 秒轮询 `/api/sol-price`，无刷新更新
- 移动端折叠为仅显示 SOL 价格 + 涨跌幅
- 点击 Ticker 跳转至行情详情页

### 策略二：加密货币词汇表系统

> **优先级：P1 | 工期：1–2 周 | 影响：极高 SEO | 开发量：低**

词汇表是获取 SEO 流量最快的方式。每一个词条都是一个可被搜索引擎索引的独立页面，"XXX 是什么意思"类长尾词搜索流量极大。

#### 2.1 词条数据结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `term` | string | 词条名（英文） |
| `term_zh` | string | 词条名（中文） |
| `definition` | text | 200 字以内简明解释 |
| `example` | text | 实际使用场景举例 |
| `category` | enum | DeFi / NFT / Security / ... |
| `related` | array | 相关词条内链 |
| `solana_specific` | bool | 是否 Solana 专属 |

#### 2.2 词条分类规划（共 200 词）

| 分类 | 数量 | 示例 |
|------|------|------|
| Solana 专属概念 | 40 | PoH, Sealevel, Turbine... |
| DeFi 基础术语 | 40 | AMM, TVL, APY, Liquidity Pool... |
| NFT 与数字资产 | 30 | Floor Price, Royalty, Metadata... |
| 交易与钱包 | 30 | Gas, Slippage, Seed Phrase... |
| 安全与风险 | 25 | Rug Pull, Phishing, Honeypot... |
| 社区与文化 | 20 | WAGMI, NGMI, Degen, HODL... |
| 技术底层 | 15 | Smart Contract, RPC, Block... |

#### 2.3 SEO 关键策略

每个词条生成独立 URL：`/glossary/proof-of-history`

页面 title 格式：
```
什么是 Proof of History（PoH）？| BuySolanas 词汇表
```

这种格式精准命中"什么是 XXX"类搜索意图，是 CoinGecko Glossary 页面获取大量搜索流量的核心原因。

#### 2.4 词汇表页面功能需求

- 首字母 A-Z 导航 + 分类筛选（DeFi / NFT / Solana...）
- 全文搜索，输入即时过滤，无需跳转
- 每个词条页有"相关词条"内链，提升 SEO 深度
- "Solana 专属"标签特殊标注，体现差异化
- 词条页底部推荐相关课程，引导用户进入学习系统
- 支持用户"收藏词条"（需登录），后续可做个人词汇本

### 策略四：Newsletter 订阅系统

> **优先级：P3 | 工期：1 周 | 影响：高留存率 | 开发量：极低**

CoinGecko 靠 60 万+ 订阅者建立极强的用户留存。Newsletter 是成本最低、ROI 最高的用户触达工具，尤其适合加密领域。

#### 4.1 每周周报内容结构

1. 本周 SOL 价格表现回顾
2. Solana 生态 3 条重要新闻
3. 本周推荐学习：1 个词条 + 1 篇课程
4. 本周链上数据亮点（TVL 变化 / 新项目）
5. 社区问答精选（Q&A）
6. 下周值得关注的事件

#### 4.2 自动化欢迎邮件序列

| 时间节点 | 邮件内容 |
|---------|---------|
| Day 0 | 欢迎 + 平台功能介绍 |
| Day 1 | 推送"新手必读"第一课 |
| Day 3 | 推送词汇表入门 10 词 |
| Day 7 | 引导创建 Solana 钱包 |
| Day 14 | 引导加入 Discord 社区 |
| Day 30 | 成就回顾 + 进阶课程推荐 |

#### 4.3 工具推荐

推荐使用 **Resend**（现代邮件 API，免费 3,000 封/月）+ **React Email**（用 React 写邮件模板）。

相比 Mailchimp 的优势：
- 开发体验更好，更易与 Next.js 集成
- 代码与网站同仓库管理
- 订阅表单使用 Next.js API Route 实现，数据存入自己的数据库，不依赖第三方

#### 4.4 订阅入口位置规划

- **Hero 区副 CTA**（主 CTA 是"开始学习"，次 CTA 是"订阅周报"）
- **课程文章底部弹出框**（阅读进度 80% 时触发）
- **词汇表页面右侧固定卡片**
- **Footer 全局订阅区域**
- **退出意图弹窗**（用户将鼠标移至页面顶部时触发）

## 5. 技术栈选型

针对中小团队/独立开发者的务实选型，兼顾开发效率与未来可扩展性。

### 前端框架

| 技术 | 用途 | 备注 |
|------|------|------|
| **Next.js 14+** | 全栈框架 | App Router，SSG/SSR 兼顾 SEO |
| **Tailwind CSS** | 样式系统 | 快速开发，维护方便 |
| **TypeScript** | 类型安全 | 减少运行时错误 |
| **Framer Motion** | 动画效果 | 页面过渡与微交互 |

### 后端 & 数据库

| 技术 | 用途 | 备注 |
|------|------|------|
| **Next.js API Routes** | 后端接口 | 无需独立后端服务 |
| **Supabase** | 数据库 + Auth | PostgreSQL + 内置用户系统，免费计划足够 |
| **Redis (Upstash)** | 行情数据缓存 | Serverless Redis，按用量计费 |
| **MDX** | 课程内容管理 | Markdown 扩展，支持嵌入 React 组件 |

### 外部服务

| 服务 | 用途 | 费用 |
|------|------|------|
| **CoinGecko API** | 行情数据 | 免费（30次/分钟） |
| **Resend** | 邮件发送 | 免费（3,000封/月） |
| **Vercel** | 部署 + CDN | 免费（Hobby 计划） |
| **Plausible** | 隐私友好统计 | 付费（$9/月） |

### 技术选型理由

> **Next.js + Supabase + Vercel** 被称为"全栈铁三角"：Next.js 处理前后端（API Routes 替代独立 Express），Supabase 提供开箱即用的 PostgreSQL + Auth + 实时订阅，Vercel 零配置部署。对于 1–2 人团队，这是开发效率最高的组合。Supabase 免费计划足够支撑 Phase 1–2。
