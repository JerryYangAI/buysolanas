策略 01 / 五大核心

实时行情 Ticker

参照 CoinGecko 顶部价格 bar，接入实时 SOL 数据，让用户每次访问都能看到最新行情，提升网站的"活跃感"与可信度。

01
SOL 实时行情 + 生态数据看板
高影响 低开发量 2–3 周 P0 优先
▼
Ticker Bar 显示字段
  • SOL/USDT 实时价格
  • 24h 涨跌幅（颜色区分）
  • 24h 交易量
  • 市值（Market Cap）
  • Solana TPS 实时值
  • 全球加密市值总量
行情详情页内容
  • K线图（1D/7D/1M/1Y）
  • 持仓地址数量趋势
  • SOL 生态 Top 10 代币
  • 质押 APY 实时数据
  • Solana 验证节点数
  • 历史价格与关键事件
数据来源方案
首选 CoinGecko Free API（每分钟 30 次调用，免费），备选 Birdeye API（Solana 专属链上数据）。生产环境建议在后端做缓存，每 30 秒刷新一次，避免前端直接调用导致限流。
// CoinGecko API 调用示例（Python 后端缓存层） import requests, redis, json def get_sol_price(): r = redis.Redis() cached = r.get('sol_price') if cached: return json.loads(cached) url = "https://api.coingecko.com/api/v3/simple/price" params = { "ids": "solana", "vs_currencies": "usd,cny", "include_24hr_change": "true", "include_market_cap": "true", "include_24hr_vol": "true" } data = requests.get(url, params=params).json() r.setex('sol_price', 30, json.dumps(data)) # 缓存 30 秒 return data
前端实现要点
  • Ticker bar 固定在 nav 下方，sticky 定位
  • 价格涨跌用绿色/红色 + 上下箭头区分
  • 每 30 秒轮询 /api/sol-price，无刷新更新
  • 移动端折叠为仅显示 SOL 价格 + 涨跌幅
  • 点击 Ticker 跳转至行情详情页
策略 02 / 五大核心

加密货币词汇表系统

词汇表是获取 SEO 流量最快的方式。每一个词条都是一个可被搜索引擎索引的独立页面，"XXX是什么意思" 类长尾词流量极大。

02
200+ 词条 · A-Z 索引 · SEO 优化词汇表
极高 SEO 低开发量 1–2 周 P1 优先
▼
词条数据结构
字段名类型说明
termstring词条名（英文）
term_zhstring词条名（中文）
definitiontext200字以内简明解释
exampletext实际使用场景举例
categoryenumDeFi/NFT/Security/...
relatedarray相关词条内链
solana_specificbool是否 Solana 专属
词条分类规划（200词）
  • Solana 专属概念 ×40（PoH, Sealevel...）
  • DeFi 基础术语 ×40（AMM, TVL, APY...）
  • NFT 与数字资产 ×30
  • 交易与钱包 ×30（Gas, Slippage...）
  • 安全与风险 ×25（Rug Pull, Phishing...）
  • 社区与文化 ×20（WAGMI, Degen...）
  • 技术底层 ×15（Smart Contract...）
SEO 关键策略
每个词条生成独立 URL：/glossary/proof-of-history。页面 title 格式："什么是 Proof of History（PoH）？| BuySolanas 词汇表"。这种格式精准命中"什么是XXX"类搜索意图，是 CoinGecko Glossary 页面获取大量搜索流量的核心原因。
词汇表页面功能需求
  • 首字母 A-Z 导航 + 分类筛选（DeFi/NFT/Solana...）
  • 全文搜索，输入即时过滤，无需跳转
  • 每个词条页有"相关词条"内链，提升 SEO 深度
  • "Solana 专属"标签特殊标注，体现差异化
  • 词条页底部推荐相关课程，引导用户进入学习系统
  • 支持用户"收藏词条"（需登录），后续可做个人词汇本

策略 04 / 五大核心

Newsletter 订阅系统

CoinGecko 靠 60 万+订阅者建立极强的用户留存。Newsletter 是成本最低、ROI 最高的用户触达工具，尤其适合加密领域。

04
每周 Solana 周报 + 自动化邮件序列
高留存率 极低开发 1 周 P3 优先
▼
每周周报内容结构
  • 本周 SOL 价格表现回顾
  • Solana 生态 3 条重要新闻
  • 本周推荐学习：1 个词条 + 1 篇课程
  • 本周链上数据亮点（TVL变化/新项目）
  • 社区问答精选（Q&A）
  • 下周值得关注的事件
自动化欢迎邮件序列
  • Day 0：欢迎 + 平台功能介绍
  • Day 1：推送"新手必读"第一课
  • Day 3：推送词汇表入门 10 词
  • Day 7：推送 Solana 钱包创建指南
  • Day 14：引导加入 Discord 社区
  • Day 30：成就回顾 + 进阶课程推荐
工具推荐：Resend + React Email
推荐使用 Resend（现代邮件 API，免费 3000 封/月）+ React Email（用 React 写邮件模板）。相比 Mailchimp，Resend 开发体验更好、更易与 Next.js 集成，且代码与网站同仓库管理。订阅表单使用 Next.js API Route 实现，数据存入自己的数据库，不依赖第三方表单服务。
订阅入口位置规划
  • Hero 区副 CTA（主 CTA 是"开始学习"，次 CTA 是"订阅周报"）
  • 每篇课程文章底部的订阅弹出框（阅读进度 80% 时触发）
  • 词汇表页面右侧固定订阅卡片
  • Footer 全局订阅区域
  • 退出意图弹窗（用户将鼠标移至页面顶部时触发）

为什么选 Next.js + Supabase + Vercel？
这三者合称"Next.js 全栈铁三角"：Next.js 处理前后端（API Routes 替代独立 Express 服务），Supabase 提供开箱即用的 PostgreSQL + Auth + 实时订阅，Vercel 零配置部署。对于 1-2 人团队，这是开发效率最高的组合。Supabase 免费计划足够 Phase 1-2 使用。
